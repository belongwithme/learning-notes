Vercel 怎么知道该执行什么？答案就在 package.json

package.json 里的 scripts 是项目提供给工具使用的命令菜单。Vercel 识别到 Astro 后，会调用其中的 build 命令。

REAL PROJECT CODE 
  "scripts": {
    "dev": "astro dev",
    "start": "astro dev",
    "build": "astro build",
    "check": "astro check",
    "preview": "astro preview",
    "astro": "astro"
  },
说人话

下面是一组可执行命令。

dev 与 start 用于本地边改边看。

build 把源文件做成可上线的网页成品。

check 在上线前检查类型与内容问题。

preview 在本地预览生产构建结果。

最后一项允许直接调用 Astro 工具。

💡
关键认知：Vercel 保存的是“每一次成品”

每次推送都会产生一个不可变的 构建产物。因此预览、晋级和回滚都可以围绕同一份成品进行，而不必现场修改服务器文件。
